# Лабораторная работа №1 – Анализ предметной области

**Предметная область:** Проектирование информационной системы представительства туристической фирмы в зарубежной стране

---

## Цель и задачи проекта

### Основная цель
Автоматизировать работу представительства туристической фирмы за рубежом: от приёма данных о туристах из России до выдачи виз, расселения, организации экскурсий, работы с грузом и формирования финансовых отчётов.

### Основные задачи
- Учитывать данные туристов (ФИО, паспорт, возраст, дети, предпочтения по жилью) и автоматически готовить документы для виз
- Управлять бронированием номеров в гостиницах и формировать списки расселения
- Организовывать запись туристов на экскурсии и передавать данные партнёрским агентствам
- Вести учёт груза: маркировка, взвешивание, расчёт стоимости упаковки и страховки
- Формировать финансовые отчёты для головной фирмы (доходы/расходы по всем статьям)
- Поддерживать гибкую систему запросов: статистика по туристам, рейсам, грузу, экскурсиям, рентабельности

---

## Основные требования

### Основная функциональность (минимум)
| Модуль | Что делает |
|--------|-----------|
| **Туристы** | Хранение анкет, категоризация (отдых / груз / дети), история поездок |
| **Документы** | Генерация визовых пакетов, списков для таможни и аэропорта |
| **Гостиницы** | Бронирование, учёт занятых номеров, отчёты по загрузке |
| **Экскурсии** | Каталог экскурсий, запись туристов, передача списков агентствам |
| **Груз** | Весовые ведомости, маркировка, расчёт итоговой стоимости отправки |
| **Финансы** | Учёт всех операций, формирование сводных отчётов по группе/периоду |
| **Отчётность** | 15+ типовых запросов: от статистики по туристам до рентабельности представительства |

### Дополнительные требования и ограничения
- Веб-интерфейс (доступ с рабочих мест представительства)
- Защита персональных данных (паспорта, визы) – шифрование, разграничение прав
- Экспорт отчётов в PDF/Excel для печати и отправки в головной офис
- Поддержка русского и локального языка интерфейса
- Автосохранение и резервное копирование данных
- Адаптивный дизайн (работа с планшетов/ноутбуков в аэропорту и гостиницах)

## Целевая аудитория проекта

| Кто | Зачем им система |
|-----|-----------------|
| **Менеджеры представительства** | Быстро оформлять визы, бронировать жильё, решать вопросы в аэропорту |
| **Логисты/складские работники** | Учитывать груз, готовить ведомости, контролировать отправку |
| **Финансовый отдел** | Собирать данные по расходам/доходам, готовить отчёты для России |
| **Головной офис (РФ)** | Получать актуальную статистику и финансовые сводки онлайн |
| **Партнёры (гостиницы, агентства)** | Получать структурированные списки туристов и заказов (через ограниченный доступ) |

> *Дети и сами туристы не являются пользователями системы – все операции за них выполняют сотрудники.*

## Основные хранимые сущности

```
Турист
   ├─ ФИО, пол, возраст, паспортные данные
   ├─ Категория: «отдых» / «груз» / «ребёнок»
   ├─ Предпочтения по гостинице
   └─ Связь с группой и рейсом

Группа туристов
   ├─ Состав (список туристов)
   ├─ Рейс (дата, номер, тип самолёта)
   ├─ Даты прилёта/вылета
   └─ Статус (формирование / в пути / завершена)

Гостиница
   ├─ Название, адрес, категория
   ├─ Доступные номера (тип, вместимость)
   └─ История бронирований

Бронирование
   ├─ Группа/турист → гостиница
   ├─ Даты проживания, номер
   └─ Статус (подтверждено / ожидание)

Виза
   ├─ Турист, статус, дата выдачи, срок действия
   └─ Примечания от миграционной службы

Экскурсия
   ├─ Название, описание, дата, стоимость
   ├─ Агентство-исполнитель
   └─ Ограничения (возраст, физподготовка)

Запись на экскурсию
   ├─ Турист → экскурсия
   └─ Статус (записан / оплачен / отменён)

Груз
   ├─ Турист-владелец
   ├─ Кол-во мест, вес, объёмный вес
   ├─ Стоимость упаковки, страховки, итог
   └─ Статус (принят / на складе / отправлен)

Рейс
   ├─ Номер, дата, маршрут
   ├─ Тип самолёта (пассажирский / грузовой / комби)
   └─ Загрузка: места, груз, объёмный вес

Финансовая операция
   ├─ Тип (доход/расход), статья (отель, виза, экскурсия...)
   └─ Сумма, валюта, дата, привязка к группе/туристу

Отчёт
   ├─ Период, тип (финансовый / статистика / груз)
   ├─ Параметры фильтрации (категория туристов, рейс, гостиница...)
   └─ Сгенерированные данные (в виде структуры для экспорта)
```

> Данные атрибуты и сущности являются предварительными. Возможны изменения.
